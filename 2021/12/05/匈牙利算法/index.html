Hungarian Algorithm

The Hungarian matching algorithm , also called the Kuhn-Munkres algorithm, is a O(∣V∣3)O\big(|V|^3\big)O(∣V∣3)algorithm that can be used to find maximum-weight matchings in bipartite graphs , which is sometimes called the assignment problem . A bipartite graph can easily be represented by an adj matrix, where the weights of edges are the entries. Thinking about the graph in terms of an adjacency matrix is useful for the Hungarian algorithm.

The Hungarian Algorithm for Graphs

Given: the labeling lll, an equality graph Gl=(V,El)G_l=(V, E_l)Gl​=(V,El​), an initial matching MMM in GlG_lGl​, and an unmatched vertex u∈Vu \in Vu∈V and u∉Mu \notin Mu∈/M

Augmenting the matching

  1. A path is augmenting for MMM in GlG_lGl​ if it alternates between edges in the matching and edges not in the matching, and the first and last vertices are free vertices, or unmatched, in MMM. We will keep track of a candidate augmenting path starting at the vertex uuu.

  2. If the algorithm finds an unmatched vertex vvv, add on to the existing augmenting path ppp by adding the uuu to vvv segment.

  3. Flip the matching by replacing the edges in MMM with the edges in the augmenting path that are not in MMM (in other words, the edges in El−M)\left.E_l-M\right)El​−M).

Improving the labeling

  1. S⊆XS \subseteq XS⊆X and T⊆YT \subseteq YT⊆Y, where SSS and TTT represent the candidate augmenting alternating path between the matching and the edges not in the matching.
  2. Let Nl(S)N_l(S)Nl​(S) be the neighbors to each node that is in SSS along edges in ElE_lEl​ such that Nl(S)={v∣∀u∈S:(u,v)∈El}N_l(S)=\{v \mid \forall u \in S:(u, v) \in E_l\}Nl​(S)={v∣∀u∈S:(u,v)∈El​}.
  3. If Nl(S)=TN_{l}(S)=TNl​(S)=T, then we cannot increase the size of the alternating path (and therefore can't further augment), so we need to improve the labeling.
  4. Let δl\delta_{l}δl​ be the minimum of l(u)+l(v)−w(u,v)l(u)+l(v)-w(u, v)l(u)+l(v)−w(u,v) over all of the u∈Su \in Su∈S and v∉Tv \notin Tv∈/T.
  5. Improve the labeling lll to l′l^{\prime}l′ :

  • If r∈Sr \in Sr∈S, then l′(r)=l(r)−δll^{\prime}(r)=l(r)-\delta_{l}l′(r)=l(r)−δl​,

  • If r∈Tr \in Tr∈T, then l′(r)=l(r)+δll^{\prime}(r)=l(r)+\delta_{l}l′(r)=l(r)+δl​.

  • If r∉Sr \notin Sr∈/S and r∉Tr \notin Tr∈/T, then l′(r)=l(r)l^{\prime}(r)=l(r)l′(r)=l(r).

    l′l^{\prime}l′ is a valid labeling and El⊂El′E_{l} \subset E_{l^{\prime}}El​⊂El′​.

Putting it all together: The Hungarian Algorithm

  1. Start with some matching MMM, a valid labeling lll, where lll is defined as the labelling ∀x∈X,y∈Y∣l(y)=0,l(x)=max⁡y∈Y(w(x,y))\forall x \in X, y \in Y \mid l(y)=0, l(x)=\max _{y \in Y}(w(x, y))∀x∈X,y∈Y∣l(y)=0,l(x)=maxy∈Y​(w(x,y)).
  2. Do these steps until a perfect matching is found (when MMM is perfect):
  • (a) Look for an augmenting path in MMM.
  • (b) If an augmenting path does not exist, improve the labeling and then go back to step (a).
